@redirect v2 documentation

Usage

The basic usage of ?redirect v2 for accounts.aimedtuba.com is simple. Just add a ?redirect param to the end of your statement.

https://accounts.aimedtuba.com/login?@redirect=v2:YOUR_URL

To add in any parameters, include &, and then $params, separated by commas for example:

https://accounts.aimedtuba.com/login?@redirect=v2:YOUR_URL&$params=username,email

Then, you much name the params’ content by using $PARAM-NAME, for example:

https://accounts.aimedtuba.com/login?@redirect=v2:YOUR_URL&$params=username,email&$username=aimedtuba&$email=admin@aimedtuba.com

If you want to replace a value with a login, username, or email in the content use $login/username/email. For example:

https://accounts.aimedtuba.com/login?@redirect=v2:YOUR_URL&$params=username,email&$username=$username&$email=$email

Examples

Redirect back to dummy.com:

https://accounts.aimedtuba.com/login?@redirect=v2:https://www.dummy.com

Redirect back to dummy.com with example param:

https://accounts.aimedtuba.com/login?@redirect=v2:https://www.dummy.com&$params=exmaple

->

https://www.dummy.com?example

Redirect back to dummy.com with username, email, and login:

https://accounts.aimedtuba.com/login?@redirect=v2:https://www.dummy.com&$params=username,email,login&$username=$username&$email=$email&$login=$login

->

https://www.dummy.com?username=aimedtuba&email=admin@aimedtuba.com&login=_no_chance
